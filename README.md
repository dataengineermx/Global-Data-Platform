# Global-Data-Platform

Imagine that you received alerts on KPIS that matters in your company! 

This stream data (earthquakes) in customized Regions that can be parametrized by coordinates (latitud, longitud, altitud and magnitude). Data is orchestrated in raw data or bronce layer, silver and gold layer avoiding data redudancy, Data is stored in a Postgress DB and credentials are NOT exposed in code to ensure data security.. The project is organized in such a way that code, functions, transformations and pipelines can be reusable. 

Historical information serves to predict behavior, identify patterns, anomalies, seasonal behaviour, be prepared for an event that you know the consecuenses.

Just think about: 

- What happend if you DO NOT receive an alert about a big magnitude earthquake?  Data Availability

- What happend if one of the monitor stations do not work? Data distribution, Clusterization, DRP

- How do you ensure monitor stations are working and sending expected information? DAta Observability

- What happend if you got the alert for an event that do not happend? Data Quality

- What happend if your get wrong an big magnitide alert of an earthquake that happend in another continent? Business rules and Data Quality Controls

- What happend if you received an alert 2 minutes late? Data Ontime, real time replication

- How many people is impacted because of the event? Data Analytics and Dashboards

- What happend if someone get access to the monitor station information, do you know what could be the impact? Data Governance and Data Security
  
Now transfer similar questions to your business. Are you working to solve and minime the impact?

Imagine that you can now complement your data with context about how many people (segmented by age) is going to be impacted, schools, businesses by industries, socio economic data and more.

This IS NOT  solution to predict earthquakes is to demostrate, how to transform data into actionable insights that helps your company to take decisions.
